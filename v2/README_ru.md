# NSAI v2

**Граф соседств слов с классификацией связей по контекстам.** Independent research, draft v0.2.

NSAI v2 строит из текста граф непосредственных соседств между словами и хранит, в каком предложении возникло каждое ребро. На запрос вида «связаны ли A и B» система находит пути по графу и классифицирует их: **утверждение** — все рёбра пути лежат в одном контексте (так и сказано в тексте), **рассуждение** — рёбра пути пересекают границы предложений (склейка через несколько источников). Никакого синтаксического разбора, никаких POS-тегов, никакой лемматизации. Любой язык, любая последовательность символов.

> **Объяснимость встроена в архитектуру, не прикручена постфактум.** Каждый ответ указывает на конкретные предложения корпуса, из которых он получен. Нет внутреннего состояния скрывающего логику — граф и ответ сделаны из одних и тех же наблюдений. Это объяснимость принципиально иного класса, чем визуализация attention или просьба к чёрному ящику рассказать о своих причинах постфактум.

## Что это

- Дискретное хранилище наблюдений из текста — никакой компрессии в веса, никакого обучения.
- Single-author research project в продолжение v0.1, написан с нуля после переформулировки задачи.

## Что это не

- Это не парсер языка. Граф не знает о морфологии, синтаксисе, ролях.
- Это не верификатор утверждений. Граф не различает «Кот ест рыбу» от «Рыбу ест кота» на ширине 1 (см. границу системы ниже).
- Это не замена LLM. Решает другую задачу: discovery связей с прямой ссылкой на источник.

## Архитектура в одном параграфе

Текст разрезается на предложения; каждое предложение — отдельный контекст. Внутри предложения для каждой пары непосредственных соседей записывается ребро в `adj[a][b]` (двусторонне) и индексируется в обратной таблице `edge_ctx[(a,b)] → {ctx_ids}`. Базовая операция одна — `of(w)`, соседи слова. Поверх неё строится `paths(a, b, max_w)` — все простые пути ограниченной длины (DFS с backtracking). Каждый путь классифицируется через пересечение контекстов своих рёбер: если есть общий ctx — **утверждение**; иначе — **рассуждение** с пометкой узлов, через которые контекст меняется. Сила связи между парой (счётчик повторов соседства) хранится, но **в классификацию не входит**: она информативна для глаз, но не должна менять тип связи. «Слово как объект» не имеет глобальной метрики (degree, weight, частоты слова) — это сознательное архитектурное решение, подробности в [principles_ru.md](./principles_ru.md).

Полная архитектура: [principles_ru.md](./principles_ru.md) · сжатое резюме: [architecture_ru.md](./architecture_ru.md)

## Установка

Python 3.10+. Никаких внешних зависимостей.

```bash
git clone https://github.com/varshchik/nsai
cd nsai/v2
python neighbors.py
```

## Быстрый пример

```python
from neighbors import Graph

g = Graph()
g.add("В Греции живет философ Сократ.")
g.add("Философ Аристотель в Греции.")
g.add("Философ Платон живет в Афинах.")

g.explain("сократ", "философ")
# сократ ↔ философ: УТВЕРЖДЕНИЕ (1 путь)
#   ✓ [утверждение, прочность=1] сократ → философ
#       ctx0: «В Греции живет философ Сократ»

g.explain("платон", "греции")
# платон ↔ греции: РАССУЖДЕНИЕ (5 путей, 3 разных узлов смены)
#   устойчивая косвенная связь: смены через «философ»×3, «живет»×2, «в»×2
#   ~ [рассуждение, прочность=1] платон → живет → греции
#       смены через: живет
#   ...
```

«Платон в Греции» **никто прямо не сказал**, но множество маршрутов сходятся к этой связи через разные опоры — это и есть устойчивая косвенная ассоциация, помеченная как рассуждение, не как факт.

## Отношение к v1

v1 (NSAI v0.1) лежит в [v1/](../v1/) — символьная архитектура с дерево-зависимостным парсером (natasha + pymorphy3), извлекающая типизированные факты `(предикат, [участники])`. v1 и v2 решают разные задачи: v1 — структурированное извлечение фактов с ролями, v2 — обнаружение связей через соседства без типизации. Обе архитектуры остаются актуальными для своих постановок.

# Архитектура v2

Сжатый русский обзор. Полная версия — [principles_ru.md](./principles_ru.md).

NSAI v2 — граф соседств слов с классификацией связей по контекстам предложений. Принцип, лежащий в основе каждого решения: **связи живут между словами, не у слова.** Слово не имеет глобальной метрики (степень, частота, важность); оно — точка входа в обход.

## Базовый цикл

Сигнал → разрезка на предложения-контексты → запись непосредственных соседств с привязкой к ctx_id → запрос → поиск путей → классификация по пересечению контекстов.

Граф мутирует только от внешнего сигнала через `add()`. Запросы (`paths`, `classify`, `explain`) — read-only. Никаких выводов в граф не записывается; всё пересчитывается при каждом обращении.

## Что хранится

Три структуры, ничего больше:

- `adj[a][b]` — счётчик соседства пары (двусторонне).
- `contexts[ctx_id]` — исходный текст предложения.
- `edge_ctx[(a,b)]` — множество ctx_id, в которых пара возникала. Обратный индекс, нужен для O(длины пути) классификации.

Никаких типов фактов, никаких ролей, никаких меток. Слова — символы, контексты — id предложений, рёбра — пары соседних позиций.

## Базовая операция

`of(w)` — соседи слова. Всё остальное — её композиция:

- «соседи соседей»: `{k for n in of(w) for k in of(n)} - {w}`
- путь длины N: BFS / DFS многократно вызывающий `of`
- пересечение окрестностей: `set(of(a)) & set(of(b))`

«Глубины» как отдельной концепции не существует; есть параметр `max_w` — насколько далеко расширять поиск. Каждое слово на пути участвует в анализе одинаково.

## Классификация связи

Связь между двумя словами обнаруживается через путь по графу. Каждый путь классифицируется:

- **Утверждение** — длина 1 (прямое соседство) ИЛИ все рёбра пути имеют общий ctx_id. Система может показать конкретный текст-источник.
- **Рассуждение** — рёбра пути приходят из разных контекстов. Между соседними рёбрами есть **узел смены контекста**.

Различение полностью структурное: пересечение множеств `ctx_id` соседних рёбер. Никаких порогов, никаких эвристик, никакого обучения.

На запросе с несколькими альтернативными путями-рассуждениями анализируется разнообразие узлов смены:
- один и тот же узел во всех путях → слабая ассоциация через одно слово;
- разные узлы во многих путях → устойчивая косвенная связь.

## Пример: «Платон жил в Греции?»

Корпус:
```
В Греции живет философ Сократ.            ← ctx0
Философ Аристотель в Греции.              ← ctx1
Философ Платон живет в Афинах.            ← ctx2
Аристотель родился в Греции.              ← ctx4
```

Прямого утверждения «Платон жил в Греции» нет ни в одном ctx. Граф находит несколько путей:

- `платон → живет → греции` через узел смены `живет` (ребро `платон↔живет` в ctx2, ребро `живет↔греции` в ctx0)
- `платон → философ → аристотель → в → греции` через узлы `философ`, `в`
- `платон → живет → в → греции` через `живет`, `в`

Все пути — рассуждения. Узлы смены: `живет`, `философ`, `в` — три разных. Система помечает связь как «устойчивая косвенная»: маршрут к выводу есть, но прямого факта нет. Это **честный** ответ — не «да», не «нет», а «выводимо через смену контекста, никем прямо не сказано».

Сравнить с запросом «Платон жил в Афинах?» — найдётся утверждение с прямой ссылкой на ctx2.

## Сила пары — не вес слова

`adj[a][b] = N` хранится, но в классификацию не входит. Это локальный структурный факт о паре, не глобальная метрика слова. Прочность пути (min силы рёбер) — диагностический показатель, не критерий типа связи.

Сколько бы раз ни повторялась пара `платон↔ученик`, через звено `ученик↔гулял` (видено один раз в другом контексте) она не образует утверждения «платон гулял». Структурный факт о разделении контекстов перевешивает количественный факт о повторах.

## Граница системы

На ширине 1 (прямое соседство) граф **не различает направление и смену субъекта**. «Кот ест рыбу» и «Рыбу ест кот» дают одинаковые соседства.

Это специализация архитектуры, не баг. Различение ролей требует синтаксического разбора и возвращает к v1. Цена принимается ради:

- независимости от языка;
- работы с произвольными последовательностями (текст, события, действия, ноты);
- объяснимости через прямую ссылку на ctx;
- ~170 строк кода без внешних зависимостей.

## Discovery, не verification

Граф отчитывается о найденных связях с их классификацией, не выносит вердикт истинности. На запрос «P?» система показывает: «вот утверждения корпуса касающиеся P», или «вот выводы через смену контекста», или «связи в данных нет». Сам пользователь интерпретирует, какой режим уверенности применим.

Истинность определяется вводом и согласованностью с ним. Система не претендует знать больше, чем ей было показано.

## Универсальность

Граф не знает что в него подают. Один и тот же код работает на:

- тексте любого языка (без морфоанализатора);
- параллельных двуязычных текстах (перевод как соседство в общем ctx);
- последовательностях действий и событий;
- любых формальных последовательностях символов.

Граница: на флективных языках морфоформы одного слова — разные узлы. Если важно — добавляется внешний модуль кластеризации, не меняя ядра.

## Отношение к v1

v1 (NSAI v0.1) — символьная архитектура с типизированными фактами `(предикат, [участники])`, извлекаемыми UD-парсером natasha. v1 различает роли и направление, требует языкового специалиста для каждого нового языка.

v2 не различает роли. v2 работает на любом языке без подготовки.

Эти две системы решают разные задачи. v1 — для structured fact extraction. v2 — для discovery соседских связей с прямой объяснимостью. Обе остаются актуальными в репозитории.

## Статус

Черновик v0.2. Файл реализации `neighbors.py`, без внешних зависимостей. Зафиксированные направления оптимизации (persistence, streaming ingest, vocab/int ids) — инженерия, не архитектурные изменения.

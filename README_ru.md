# NSAI v2.1

**Граф соседств слов с позиционным провенансом и классификацией через окно при запросе.** Независимое исследование, рабочий рефакторинг v2.

NSAI строит граф непосредственных соседств слов в тексте и отслеживает позицию каждого вхождения в источнике. Когда задаётся вопрос о связи между двумя словами, система находит пути в графе и классифицирует их по **window-fit**: **утверждение** — существует окно заданной ширины в одном источнике, покрывающее все рёбра пути (связь локализована в одном фрагменте), **рассуждение** — такого окна нет (цепочка собрана из позиций слишком далёких друг от друга). Никакого dependency parsing, POS-тегов, лемматизации, границ предложений.

> **Объяснимость структурная, не post-hoc.** Каждый ответ указывает на конкретный позиционный фрагмент исходного текста. Никакого внутреннего состояния, скрывающего логику — граф и ответ сделаны из одних и тех же наблюдений. Это другой тип объяснимости в сравнении с визуализацией attention или просьбой к чёрному ящику пересказать своё рассуждение постфактум.

## Что это

- Хранилище дискретных наблюдений из текста — никакой компрессии в веса, никакого обучения.
- Авторский исследовательский проект, рабочий рефакторинг v2 (см. [CHANGES.md](./CHANGES.md)).

## Что это не

- Не парсер языка. Граф ничего не знает о морфологии, синтаксисе или ролях.
- Не верификатор утверждений. Граф не отличит «Кот ест рыбу» от «Рыба ест кота» при width=1.
- Не замена LLM. Решает другую задачу: открытие связей с прямой атрибуцией к источнику.

## Архитектура в одном абзаце

Текст токенизируется в непрерывный поток — знаки препинания игнорируются (это writing convention, не feature языка; в живом диалоге нет точек). Для каждой пары непосредственных соседей в token stream записывается ребро в `adj[a][b]` (двунаправленно) и индексируется по позиции в `edge_positions[(a,b)] → [(source, pos), ...]`. Единственная базовая операция — `of(w)`, соседи слова. Над ней `paths(a, b)` находит пути через weighted Dijkstra-like поиск (сильные совместные появления ранжируются выше). Каждый путь классифицируется проверкой: содержит ли какой-нибудь один source окно заданной ширины, покрывающее все рёбра пути (sliding-window minimum-cover алгоритм). Если да — **утверждение**, с цитатой `source[start..end]`; иначе — **рассуждение**, с junction nodes — узлами где соседние рёбра не помещаются в окно. Сила пары (число совместных появлений) хранится для отображения, но **не участвует в классификации**. Width — параметр запроса: узкое окно = строгая локализация, широкое = широкая ассоциация.

## Использование

### Запуск тестов

```bash
python test_classify.py
# 23 passed, 0 failed

# Или с pytest если установлен:
python -m pytest test_classify.py -v
```

### Быстрый пример

```python
from neighbors import Graph

g = Graph()
g.add("В Греции живет философ Сократ.")
g.add("Философ Аристотель в Греции.")
g.add("Философ Платон живет в Афинах.")

g.explain("сократ", "философ")
# сократ ↔ философ: УТВЕРЖДЕНИЕ (1 путь, width=6)
#   ✓ [утверждение, прочность=1] сократ → философ
#       default[3..5]: «философ сократ философ»

g.explain("платон", "греции")
# платон ↔ греции: РАССУЖДЕНИЕ (5 путей, 3 разных узлов смены, width=6)
#   устойчивая косвенная связь: смены через «философ»×3, «живет»×2, «в»×2
#   ~ [рассуждение, прочность=1] платон → живет → греции
#       смены через: живет
```

«Платон в Греции» **прямо не утверждалось**, но множество маршрутов сходятся на этой связи через разные junctions — устойчивая косвенная ассоциация, помечена как рассуждение, не как факт.

### Загрузка из файлов

```python
# Один файл:
g.load_file('corpus.txt')

# С разделителями секций для изоляции атомарных фактов
# (содержимое файла: "Сократ философ [end] Платон в Афинах")
g.load_file('atomic_facts.txt')
# создаёт sources atomic_facts:0, atomic_facts:1

# Несколько конкретных файлов:
g.load_files('socrates.txt', 'plato.txt', 'aristotle.txt')

# Вся директория:
g.load_dir('corpora/', pattern='*.txt')
```

Каждый файл получает свой source-namespace через имя файла. Узлы объединяются между файлами (одно слово = один node в графе), но позиции остаются изолированными — cross-file пути классифицируются как рассуждение по определению.

### Width как параметр запроса

Width устанавливается per query, не при построении графа:

```python
g.explain("a", "b", width=2)   # узкое окно, только адъяцентные
g.explain("a", "b", width=10)  # cross-sentence reasoning
g.explain("a", "b", width=50)  # широкая ассоциация
```

### Структурированный анализ (для UI / дальнейшей обработки)

```python
analysis = g.analyze("сократ", "философ", width=3)
# возвращает dict: { paths, assertions, reasonings, width, ... }

# Форматирование вручную:
text = g.format_analysis(analysis)
```

`analyze()` возвращает pure structured data — легко serialize в JSON, отдать во frontend, или обработать программно. `explain()` — print-wrapper вокруг `format_analysis(analyze(...))`.

### Морфологическая близость (без лемматизатора)

```python
g.morph_similarity("сократ", "сократа")  # → 0.80
g.morph_neighbors("сократ", threshold=0.5)
# → [('сократа', 0.80), ('сократе', 0.80), ...]
```

Character n-gram Jaccard similarity. Inverted index строится инкрементально — O(|n-grams(w)| × |candidates|) lookup, не O(|V|) scan. Tool только для inspection; не интегрирован в path-finding (сохраняет различие ролей между морфологическими формами).

Пример на русском потому что это рабочий корпус; сам код language-agnostic.

## Отношение к предыдущим версиям

- **v1** (NSAI v0.1) — [v1/](../v1/) — символьная архитектура с парсингом дерева зависимостей (natasha + pymorphy3), извлекающая типизированные факты `(предикат, [участники])`. ~1500 строк.
- **v2** — [v2/](../v2/) — изначальный co-occurrence граф с sentence-based контекстом. ~150 строк. Описан в [статье о негации](https://varshchik.dev/22052026_ru.pdf).
- **v2.1** — эта директория. Тот же концептуальный подход что v2, с корректировками (sliding-window classify, детерминированные paths), инженерными улучшениями (морфология, loaders, type safety), и test suite. См. [CHANGES.md](./CHANGES.md).

Все архитектуры остаются валидными для своих формулировок.
